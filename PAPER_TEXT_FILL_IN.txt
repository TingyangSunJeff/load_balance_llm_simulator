WHY THIS MATTERS:
=================

The professor's original text suggests "JFFC only" has MORE wasted memory, 
but the analysis shows it actually has LESS wasted memory (0.49 GB vs 0.85 GB).

However, "JFFC only" still performs WORSE because:
1. Lower total capacity (102 vs higher in Proposed) → more queueing
2. Less residual memory (12.67 GB vs 20.42 GB) → less room for cache
3. Higher cache per job (1.08 GB) → limits parallelism
4. Imbalanced load (c=24 on fast, c=5 on slow) → jobs spill to slow servers

So the narrative should focus on RESIDUAL MEMORY (where Proposed wins) rather 
than WASTED MEMORY (where JFFC only actually wins due to quantization effects).
